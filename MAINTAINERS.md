# Maintainers

## en-US

## Maintainer Model

This repository uses area-based maintainership. Until named maintainers are assigned, `@maintainers` in `.github/CODEOWNERS` is a placeholder team and should be replaced by the actual GitHub team or users before public collaboration.

## Areas

| Area | Responsibility |
| --- | --- |
| Core plugin | Plugin metadata, marketplace entry, README, release readiness |
| Skills | Skill frontmatter, trigger clarity, output requirements, validation guidance |
| Documentation | `en-US` and `de-DE` documentation, links, examples, process coverage |
| Scripts and validators | Python/PowerShell tools, tests, executable trust layer |
| Schemas and templates | KPI contracts, evidence, RLS, acceptance, process packs |
| Security and governance | Secrets, credential safety, tenant boundaries, release gates |

## Review Expectations

- At least one maintainer review is required for material changes.
- Security-sensitive changes require governance review.
- Generated Power BI artifacts require validation evidence.
- Substantial documentation changes require `en-US` and `de-DE`.

## de-DE

## Maintainer-Modell

Dieses Repository nutzt bereichsbasierte Maintainership. Solange keine konkreten Maintainer eingetragen sind, ist `@maintainers` in `.github/CODEOWNERS` ein Platzhalter-Team und sollte vor oeffentlicher Zusammenarbeit durch das echte GitHub-Team oder konkrete Nutzer ersetzt werden.

## Bereiche

| Bereich | Verantwortung |
| --- | --- |
| Core Plugin | Plugin-Metadaten, Marketplace-Eintrag, README, Release Readiness |
| Skills | Skill-Frontmatter, Trigger-Klarheit, Output Requirements, Validierungshinweise |
| Dokumentation | `en-US`- und `de-DE`-Dokumentation, Links, Beispiele, Prozessabdeckung |
| Scripts und Validatoren | Python-/PowerShell-Tools, Tests, ausfuehrbarer Trust Layer |
| Schemas und Templates | KPI Contracts, Evidence, RLS, Acceptance, Process Packs |
| Security und Governance | Secrets, Credential Safety, Tenant-Grenzen, Release Gates |

## Review-Erwartungen

- Wesentliche Aenderungen brauchen mindestens ein Maintainer-Review.
- Security-sensitive Aenderungen brauchen Governance Review.
- Generierte Power-BI-Artefakte brauchen Validierungsevidence.
- Wesentliche Dokumentationsaenderungen brauchen `en-US` und `de-DE`.
