# Validation and Governance

## Required Checks

- Reconcile row counts and financial totals against trusted source extracts.
- Validate every KPI numerator, denominator, date basis, exclusion, and filter behavior.
- Test RLS with representative users and denied-access cases.
- Confirm privacy levels, gateway, refresh identity, incremental refresh policy, and failure alerts.
- Document model owner, data owner, refresh SLA, certification status, and release checklist.

## Security

Treat PII, payroll, health, customer, supplier, contract, and finance data as sensitive. Use least privilege, RLS/OLS where needed, masked drillthrough fields, and approved workspaces.
