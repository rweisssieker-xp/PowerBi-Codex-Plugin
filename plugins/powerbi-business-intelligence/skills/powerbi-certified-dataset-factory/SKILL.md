# Power BI Certified Dataset Factory

Use this skill when many reports should use fewer governed semantic models.

## Workflow

1. Identify candidate domain: finance, sales, customer, material, procurement, service, HR, risk, or executive.
2. Define model ownership: business owner, data owner, Power BI owner, support owner.
3. Build certified dataset contract: facts, dimensions, measures, hierarchies, RLS/OLS, refresh, sensitivity, endorsement.
4. Define consumer contract: allowed fields, certified measures, drillthrough keys, unsupported scenarios.
5. Create migration plan from local models to certified model.

## Output Requirements

- Include dataset purpose, scope boundaries, certified measures, RLS roles, refresh SLA, lineage, and deprecation plan.
- Call out where Direct Lake, DirectQuery, import, or composite model affects self-service behavior.
