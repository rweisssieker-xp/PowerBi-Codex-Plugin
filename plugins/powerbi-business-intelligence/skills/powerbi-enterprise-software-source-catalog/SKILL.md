# Power BI Enterprise Software Source Catalog

Use this skill to select the correct source-product pack before designing extraction, data products, or reports.

## Source Product Families

- EPM/CPM/planning.
- Treasury, cash, banking.
- Tax, compliance, audit, GRC.
- ESG and sustainability.
- QM/QMS, EHS, CAPA, quality, regulatory quality.
- Customer experience, marketing, commerce.
- Subscription, revenue, billing.
- Field service, aftermarket, installed base.
- Procurement and supplier risk.
- Logistics visibility, carrier, parcel.
- WHS/WMS, warehouse automation, robotics, scanners.
- APM, observability, IT Ops.
- Data catalog, governance, MDM.
- ETL/ELT/iPaaS/integration.
- Data quality and data observability.
- Lakehouse and warehouse platforms.
- Document/content/ECM.
- OCR/IDP/invoice capture.
- Low-code, workflow, RPA.
- Identity, security, access.

## Output Requirements

- Identify product family, common vendors, source objects, integration method, Power BI grain, KPIs, risks, and validation evidence.
- Mark product-specific object names as candidates until verified in the customer tenant.
- Provide matching `en-US` and `de-DE` documentation for substantial source mappings.
