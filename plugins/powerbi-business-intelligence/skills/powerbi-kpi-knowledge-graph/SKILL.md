# Power BI KPI Knowledge Graph

Use this skill when KPI governance needs relationships, not flat catalogs.

## Node Types

- KPI, measure, business term, process, process step, source system, source object, field, report, page, owner, role, domain, data product.

## Edge Types

- defines, calculates, validates, uses, owns, conflicts-with, replaces, sourced-from, shown-on, certified-by, restricted-by.

## Output Requirements

- Include graph schema, candidate nodes/edges, conflict rules, owner model, and update cadence.
- Use the graph to identify duplicate KPIs, undocumented measures, orphaned reports, and lineage gaps.
