# Power BI Industrial Root Cause Cockpit

Use this skill when a business problem crosses multiple process steps and needs root-cause analysis.

## Root Cause Pattern

1. Define symptom KPI: margin loss, late delivery, overdue AR, stockout, downtime, scrap, forecast error, warranty cost.
2. Decompose by process step, source system, owner, time, plant, product, customer, supplier, and document status.
3. Link upstream and downstream facts through validated document flow keys.
4. Separate controllable drivers from external drivers.
5. Produce an exception table with owner, root cause, financial impact, next action, and aging.

## Common Cockpits

- Order2Cash Reliability: order block, ATP miss, delivery delay, invoice dispute, overdue cash.
- Margin Leakage: quote margin, discount, purchase price, production variance, freight, warranty, returns.
- Inventory Risk: forecast error, slow moving stock, safety stock, supplier delay, production plan variance.
- Supplier Performance: OTIF, quality defects, price variance, blocked invoices, contract leakage.
- Production Performance: OEE, downtime, scrap, rework, capacity bottleneck, order variance.
- Quality and Warranty: defect source, product family, supplier, plant, service cost, corrective action aging.

## Output Requirements

- Include symptom tree, process driver map, source tables/entities, model grain, DAX measures, report pages, and validation.
- Require business owner sign-off for root-cause categories before operationalizing actions.
