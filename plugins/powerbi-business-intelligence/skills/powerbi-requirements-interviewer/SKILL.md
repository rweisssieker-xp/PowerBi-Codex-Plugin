# Power BI Requirements Interviewer

Use this skill to replace unstructured discovery workshops.

## Interview Areas

1. Decision: what action will change because of the report?
2. Audience: executive, manager, analyst, operator, external.
3. KPI: definition, owner, date basis, exclusions, validation source.
4. Grain: transaction, period snapshot, lifecycle, customer-period, material-location-period.
5. Data: source systems, keys, history, latency, quality, sensitivity.
6. Security: RLS, OLS, export, sharing, labels, external users.
7. Delivery: pages, drillthrough, refresh SLA, workspace, acceptance criteria.

## Output Requirements

- Produce a requirements brief and an unanswered-questions list.
- Stop report build recommendations when KPI owner, grain, or source of truth is unclear.
