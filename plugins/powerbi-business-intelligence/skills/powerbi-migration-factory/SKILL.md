# Power BI Migration Factory

Use this skill when existing reporting must move into a governed Power BI target.

## Workflow

1. Inventory legacy reports, data sources, owners, KPIs, refresh process, and usage.
2. Classify migration type: lift-and-shift, rebuild, consolidate, retire, or replace with certified dataset.
3. Map source fields, calculations, filters, security, and output pages.
4. Create target semantic model, KPI catalog, report page mapping, and validation tests.
5. Run acceptance with business owner and retire legacy artifacts deliberately.

## Output Requirements

- Include source-to-target mapping, KPI parity plan, visual parity plan, validation, cutover, rollback, and retirement checklist.
