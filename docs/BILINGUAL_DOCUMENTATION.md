# Bilingual Documentation

Substantial artifacts generated by the plugin should include aligned `en-US` and `de-DE` documentation.

The detailed plugin-local standard is:

```text
plugins/powerbi-business-intelligence/docs/BILINGUAL_DOCUMENTATION_STANDARD.md
```

Templates:

```text
plugins/powerbi-business-intelligence/docs/templates/artifact.en-US.md
plugins/powerbi-business-intelligence/docs/templates/artifact.de-DE.md
```

## Required Artifact Types

Use bilingual documentation for:

- Delivery packs.
- KPI catalogs.
- Semantic model documentation.
- Report specifications.
- Data contracts.
- Quality gate evidence.
- Governance documentation.
- Training material.
- Business owner acceptance packs.
- AI/KI generated management narratives.

## Minimum Structure

Each language version should include:

- Purpose.
- Scope.
- Sources.
- KPIs.
- Model.
- Security.
- Validation.
- Operations.
- Owners.
- Open decisions.

## Translation Rules

- Keep KPI names aligned unless the enterprise has approved local terminology.
- Keep technical object names unchanged.
- Translate business explanations, acceptance criteria, risks, and operating notes.
- Use the same section order in both languages.
