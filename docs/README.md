# Documentation Folders

Repository-level GitHub documentation lives in the root and is indexed by [DOCUMENTATION.md](../DOCUMENTATION.md).

This folder contains deeper documentation that should not crowd the repository root.

## Folders

- [product](product/README.md): detailed product, process, source-system, KPI, connector, and skill documentation.
- [internal](internal/README.md): maintainer-oriented planning notes, rollout notes, bilingual documentation policy, and simulation reports.

## Rule Of Thumb

Keep root documentation concise and English-first. Move detailed product catalogs and internal planning material into this folder.
